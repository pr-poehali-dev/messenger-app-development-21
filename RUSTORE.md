# 📱 Загрузка Nova в RuStore

Эта инструкция превращает твой веб-мессенджер в настоящее Android-приложение (`.aab` файл) и помогает опубликовать его в RuStore.

> ⏱ Времени всего: ~1 час в первый раз, потом обновление за 5 минут.

---

## ✅ Что уже готово (я сделал)

- Установил Capacitor (обёртка веб-приложения в нативный Android)
- Создал `capacitor.config.ts` с настройками Nova
- Подготовил `.gitignore` для Android-файлов
- Сгенерировал иконку приложения (см. ниже как её применить)

---

## 🛠 Шаг 1. Установи нужные программы (только в первый раз)

### 1.1 Android Studio (бесплатно)
Скачай и установи: https://developer.android.com/studio

При первом запуске он сам скачает Android SDK (~3 ГБ). Просто соглашайся с предложениями.

### 1.2 JDK 17
Обычно уже идёт вместе с Android Studio. Проверить можно командой `java -version` в терминале.

---

## 🚀 Шаг 2. Собери Android-проект (выполни локально, не на poehali)

Скачай свой проект: **Скачать → Скачать код** (или подключи GitHub).

Открой папку проекта в терминале и выполни **по очереди**:

```bash
# 1. Установить зависимости (если ещё не установлены)
npm install

# 2. Собрать веб-версию
npm run build

# 3. Создать Android-проект (только в первый раз!)
npx cap add android

# 4. Сгенерировать иконки/сплеш-экраны (опционально, см. шаг 3)
# npx capacitor-assets generate --android

# 5. Синхронизировать веб-код с Android
npx cap sync android

# 6. Открыть проект в Android Studio
npx cap open android
```

После последней команды откроется Android Studio с проектом Nova.

---

## 🎨 Шаг 3. Иконка приложения (опционально, но рекомендуется)

Я сгенерировал красивую иконку: **скачай её** по ссылке ниже и сохрани в проект:

🔗 https://cdn.poehali.dev/projects/6364bfec-87ef-4e7b-8203-730d57164065/files/aaea7266-b085-4ec1-8974-ef177a44664e.jpg

1. Создай в корне проекта папку `assets/`
2. Сохрани туда файл как **`icon.png`** (минимум 1024×1024)
3. Туда же положи **`splash.png`** (можно ту же картинку, размер 2732×2732)
4. Выполни команду: `npx capacitor-assets generate --android`

Capacitor сам создаст все нужные размеры иконок (mdpi, hdpi, xhdpi, xxhdpi, xxxhdpi) и сплеш-экраны.

---

## 🔑 Шаг 4. Создай ключ подписи (только в первый раз!)

⚠️ **Очень важно:** этот ключ нужен на ВСЮ жизнь приложения. Потеряешь — не сможешь обновлять Nova в RuStore. Сохрани в нескольких местах + запомни пароли.

В Android Studio:
1. Меню **Build → Generate Signed Bundle / APK**
2. Выбери **Android App Bundle** → Next
3. Под полем "Key store path" нажми **Create new...**
4. Заполни:
   - **Key store path:** `nova-release-key.jks` (сохрани в безопасное место — НЕ в папку проекта!)
   - **Password:** придумай надёжный пароль (запиши)
   - **Key alias:** `nova`
   - **Key password:** ещё один пароль (можно тот же)
   - **Validity:** 25 лет
   - **First and Last Name:** твоё имя
   - **Organization:** Nova (или название твоей компании)
   - **City / State / Country:** свои данные
5. Нажми **OK**

---

## 📦 Шаг 5. Собери .aab файл для RuStore

В том же окне после создания ключа:
1. Убедись что выбран **release** вариант
2. Нажми **Create**
3. Подожди 2-5 минут (Android Studio собирает приложение)

Готовый файл будет здесь:
```
android/app/release/app-release.aab
```

Это и есть тот файл, который ты загружаешь в RuStore.

---

## 🏪 Шаг 6. Загрузка в RuStore

1. Зарегистрируйся как разработчик: https://www.rustore.ru/console/
2. Подтверди личность (паспорт / самозанятый / ИП — RuStore требует)
3. **Создать приложение** → загрузи:
   - **AAB-файл:** `app-release.aab` (из шага 5)
   - **Иконка:** 512×512 PNG
   - **Скриншоты:** минимум 2 (можно прямо со своего телефона)
   - **Описание:** короткое (80 знаков) и полное (4000 знаков)
   - **Возрастное ограничение:** 0+ или 12+
   - **Категория:** Социальные сети / Связь
4. Отправь на модерацию
5. Обычно одобряют за 1-3 дня

## 🔄 Как обновлять приложение в дальнейшем

Каждый раз когда хочешь обновить Nova:

1. **В `android/app/build.gradle` увеличь номер версии:**
   ```gradle
   versionCode 2        // ← было 1, стало 2 (целое число, всегда +1)
   versionName "1.0.1"  // ← было "1.0.0"
   ```

2. **Пересобери:**
   ```bash
   npm run build
   npx cap sync android
   npx cap open android
   ```

3. В Android Studio: **Build → Generate Signed Bundle/APK**
4. Используй **тот же ключ** что и в первый раз
5. Залей новый `app-release.aab` в RuStore Console

## ❓ Частые проблемы

**«SDK location not found»**
→ Android Studio → Settings → Android SDK → запомни путь → создай файл `android/local.properties` с содержимым:
```
sdk.dir=/Users/твоё_имя/Library/Android/sdk
```
(на Windows: `C:\\Users\\твоё_имя\\AppData\\Local\\Android\\Sdk`)

**«Gradle build failed»**
→ В Android Studio: меню **File → Invalidate Caches → Invalidate and Restart**

**Приложение белое / не загружается**
→ Открой `capacitor.config.ts` и проверь `webDir: "dist"`. После любых изменений делай `npm run build && npx cap sync android`.

**Хочешь чтобы приложение работало с твоим бэкендом без интернет-сборки?**
→ Раскомментируй блок `server` в `capacitor.config.ts` и укажи URL опубликованного сайта. Тогда приложение всегда будет грузить актуальную версию из сети.

---

## 📌 Важно знать

- **appId** в `capacitor.config.ts` (`ru.nova.messenger`) — менять нельзя после первой публикации в RuStore
- RuStore принимает только `.aab` (Android App Bundle), не `.apk`
- Минимальная версия Android по умолчанию — 7.0 (Android API 24)
- Вес итогового файла обычно 5-15 МБ
